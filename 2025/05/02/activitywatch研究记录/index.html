activitywatch研究记录

前言

最近在研究 activitywatch 这款软件, 以下简称为 aw。它是用来追踪记录个人使用电脑的情况的开源软件, 在 github 上也有几千的 star. 在基础的层面上,
aw 支持追踪记录窗口和进程的使用情况, 但也可以自己扩展用来记录自己需要的信息, 如浏览器上的使用记录了, 在听音乐的情况的之类的.
开源免费, 还能自定义, 相较于收费的软件来说还是很划算的. 但使用的难度就上来了. 所以就打算把自己使用的感想了, 操作方法之类的记录下来, 万一那天忘了啥, 还可以回顾一下. 我用的是 windows 版本的.

基础介绍

首先, 软件本身是可以直接去 github 上下载的, 同时还有自己的官方网站和官网文档可以查看, 但都是英文的, 目前还没有见到中文的翻译版本.
软件在启动之后就自动最小化到托盘了, 就已经开始进行记录了. 软件在这方面分为了两部分, 一部分是用来收集数据的记录器, 正式名称叫 watcher, 另一部分就是用来存储和可视化这些追踪数据的 serve. 我们在托盘图标上右键, modules 选项下面的是所有存在的模块, 带有 watcher 字段的是存在的追踪器, 打对勾是正在使用的. 还是右键, 点击 open dashboard 可以打开统计窗口.
首先进去是欢迎界面, 这个可以在设置里面修改. 目前软件是全英的. 在左上角, 有 activity 点击进去就可以看到当前的一些追踪记录. 这些追踪记录可以在设置中通过关键词渲染出不同的颜色, 让数据更加好看. timeline 则是以时间流的方式显示数据. 余下的那个 stopwatch 就是用来在指定情况下停止追踪记录使用的

watcher 的基础使用

使用 aw 这个软件主要目的还是追踪记录自己, 但不同的人有不同的记录的需求, 因此可以使用不同的 watcher 在满足自己的需求.
自带的 watcher 有三个, 分别是:

  1. aw-watcher-afk : 用来追踪使用者是不是在使用电脑, 默认情况下如果键盘和鼠标有三分钟没有动过, 就会记录人已经不在电脑旁边了
  2. aw-watcher-window: 记录了当前那个窗口在被使用, 标题是什么, 如果是浏览器的话还会记录 url
  3. aw-watcher-input - 跟踪按键次数和鼠标移动的距离。

除了这些自带的, 有时候是我们也会想追踪一些其他的数据, 这就需要添加一些自定义的 watcher, 也不一定需要自己写, 在 aw 的官网上就推荐了一些第三方的 watcher.
在 github 上找到需要的 watcher 之后下载下来, 解压为文件夹. 到 aw 的安装目录, 可以看到与原生的 watcher 同名的文件夹, 直接复制过来, 和原生的 watcher 同名的文件夹放在一起就可以了. 然后重启一下 aw ,就可以再 modules 下面看到新装的 watcher 了, 单击启动使用.
如果想要让 aw 启动的时候就自动启动 watcher 就需要到 C:\Users\~\AppData\Local\activitywatch\activitywatch\aw-qt 下面, 有一个 aw-qt. toml 文件, 在 autostart_modules = ["aw-server", "aw-watcher-afk", "aw-watcher-window"] 中添加自己希望软件启动后就自动启动的 watcher 的名字就可以了

## activitywatch的各组件

aw大致可以分为三部分, watcher, serve

之前介绍的watcher是用来收集信息的部分，自然也就有汇集和存储信息的部分aw-serve,同时还有将信息可视化出来的面板部分aw-webui

如何做一个自定义的watcher

每个人有自己不同的需求，往往有时候在网上是找不到适合自己的watcher的，这时候就要自己写了。

aw支持python和rust用来开发wa,我习惯用python就用python来进行了

官方的网站虽然提到了一些watcher的开发

给了一个极其简易的watcher，但也就仅止于此了，至于如何进行测试了，应该怎么做才能让自己写的代码运行起来，代码不全能不能行之类的就很少提及

所幸的是，有还有api文档的存在以及其他人的文件可以参考

首先，如果要开发watcher建议不要直接用官方给的例子，而是自定义一个watcher的类

根据我参考他人的开源文件得来的经验，目录一般情况下是如下的排布

其中src中的watcher是核心，里面需要引用aw中的event以及clinet两个类

watcher收集来的数据要封装在一个event中，然后发送到serve去。在serve中用于存储数据的结构叫bucket。一般来说，一个watcher对应一个bucket。

我们需要初始化一个cliner类，通过cliner通过的方法来在serve中创建，检索修改bucket，并向其中发送自己存储到的数据

我们可以自定义watcher类用来向serve发送收集到的数据，而收集数据看情况是否需要自定义类来进行处理

actitywatch的 API

client

client 中有两个类 ActivityWatchClient 以及 RequestQueue .
ActivityWatchClient 是一个客户端类, 用来和 serve 端进行通信, 创建修改bucket 以及 event. RequestQueue 是一个服务类, 用于 client 于 serve 之间暂时连接不上时存储发送的 event

ActivityWatchClient

ActivityWatchClient 是对 aw-server 的REST API 的一个简易的包装, 其实质上还是使用的 requests 库通过本地的端口进行通信. 因此, 在初始化的时候我们可以有选择性的向里面传入 host, port, 以及 protocol, 用来对其进行修改
