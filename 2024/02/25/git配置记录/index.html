git配置记录

发表于2024-02-25|更新于2024-02-25|计算机
|阅读量:

前言

由于系统原因,我需要定期的重装系统.为了方便装完系统之后对git进行配置,便留一个记录以供参考

下载与安装

下载是在git的官网上下载下来的,官网的地址在此：https://git-scm.com/download

另外，官网上提供有一份git的使用指南，叫pro git,可以直接下载,是纯英文的,国内应该有汉化版

对于git如果没有特殊的配置需要,直接按默认安装即可,如果担心C盘的容量,可以改一下路径

配置本地账户

git的配置可以分为两个部分:

  1. 配置本地的账户
  2. 与在github上的账号相关联

配置本地账户

在本地需要配置的是首先是个人的名称和邮箱,这些信息要在版本提交的过程中作为标识提交上去,而其他的可以稍后在进行设置

通过git config --list --show-origin可以列出配置文件及其配置信息

配置GitHub

首先要有一个正常使用的github账号,这一步就不再赘述
拥有一个账号后,需要设置ssh key,在连接到github上的远端仓库是github会对ssh key进行验证.
在本地电脑上打开git bash然后输入以下命令,将your_email@example.com替换为你自己的邮箱

在输入命令之后,可以在c:/Users/YOU/.ssh文件夹下找到生成的文件,一共有两个id_ed25519和id_ed25519.pub,后缀是pub的文件要用于后续的配置,其中的内容要填入github的设置中

在本地的内容生成完毕之后,可以打开github的设置页面,左侧边栏有SSH and GPG keys的选项,进入其中,点击new ssh key ,输入一个自定义的title,然后将id_ed25519.pub文件中的全部内容复制到Key中
如此一来,基本的设置就完成.
